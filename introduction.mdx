## Overview

Welcome to Godfred Osei's documentation platform. You use this space to organize, manage, and publish all your project documentation in one centralized location. Godfred Osei simplifies documentation workflows by providing intuitive tools for writing, versioning, and collaborating on docs. Our mission is to empower developers and teams to create high-quality, accessible documentation without the hassle of complex setups.

Whether you document APIs, user guides, or internal wikis, Godfred Osei handles it all with markdown support, live previews, and seamless integrations.

## Quick Start

Get up and running in minutes. Follow these steps to create your first documentation page.

<Steps>
  <Step title="Sign Up" icon="user-plus">
    Create your free account at `https://godfredosei.com/signup`.
  </Step>
  <Step title="Create a Workspace" icon="folder-plus">
    Set up a new project space for your docs.
  </Step>
  <Step title="Write Your First Page" icon="edit">
    Use the editor to add markdown content. Here's a simple example:

    ````markdown
    # My First Doc

    Welcome to your documentation!

    ```javascript
    console.log("Hello, Godfred Osei!");
    ```
    ````
  </Step>
  <Step title="Publish" icon="globe">
    Hit publish to make your docs live at a custom URL.
  </Step>
</Steps>

<Callout kind="tip">
  Pro tip: Integrate with GitHub for automatic syncing of your markdown files.
</Callout>

## Brand Guidelines

Maintain consistency across your docs by following these guidelines.

<Expandable title="Primary Brand Color" default-open="true">
  Use our primary blue `#3B82F6` for headings, links, and accents.

  ```css
  :root {
    --primary-color: #3B82F6;
  }

  h1, h2, a {
    color: var(--primary-color);
  }
  ```
</Expandable>

<Expandable title="Typography and Icons">
  Stick to system fonts and Lucide icons for a clean look. Avoid custom fonts unless necessary.
</Expandable>
